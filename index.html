How This Report Is Generated

1. Parse FCA news RSS feed for crypto-related articles; scan articles for new FRN numbers

2. Search ~40 crypto-related terms via FCA Register API to discover new FRNs

3. Merge RSS discoveries + keyword search + known FRN seed list (61 firms)

4. For each FRN, fetch firm details, trading names, and address from FCA API

5. Filter for MLRs Status = "MLRs Registered" (cryptoasset businesses)

6. Diff against previous snapshot for new/removed firms

7. Generate this HTML and push to GitHub Pages

Note: The FCA API cannot list all cryptoasset firms (no filter-by-MLRs-status endpoint). The authoritative source is the predefined=CA register page, but it's a Salesforce SPA that can't be scraped server-side. The workflow uses three discovery channels: (1) RSS feed scanning for FRNs in news articles, (2) keyword search, and (3) a manually maintained seed list. The seed list is periodically verified against the full register.

How This Report Is Generated

1. CBI publishes the Register of Authorised CASPs as a PDF (no API available)

2. Seed data is parsed from the PDF (entity name, reference, services, passporting countries)

3. Diff against previous snapshot for new/removed CASPs

4. Generate this HTML and push to GitHub Pages

Note: CBI data is manually seeded from official PDF registers. When CBI publishes updated PDFs, new data is extracted and the seed is refreshed.
